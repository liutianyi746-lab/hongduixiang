# 资料导入

## 导入前

先提醒用户确认自己有权使用聊天、照片和社交资料。未经授权的私密照片、账号内容、定位或秘密获取的数据不得导入。

默认只读取并生成必要派生档案，不复制原文件。若用户要求保留原始文件，说明实际本地路径和删除方式。

## WeFlow TXT

已验证的文本格式：

```text
2025-04-04 21:10:39 '发送者'
消息正文

2025-04-04 21:11:00 '另一位发送者'
下一条消息
```

正文可跨行。导入后必须让用户确认“我”和女朋友分别对应哪个发送者，再进行人物或语气提取。

系统提示、撤回提示和加好友提示不进入证据消息。`[表情包]`、`[图片]`、`[视频]` 等只汇总类型和频率，不把占位正文写入证据输出。

## JSON 与 JSONL

当前解析器接受顶层消息数组，或包含 `messages`、`records`、`data` 数组的对象。支持字段：

- 时间：`timestamp`、`time`、`datetime`、`createTime`；
- 发送者：`sender`、`senderName`、`name`、`nickname`；
- 正文：`content`、`text`、`message`、`msg`；
- 可选类型：`type`、`msgType`。

ChatLab、Arkme JSON 或其他嵌套结构只有在获得脱敏样例并验证映射后才能宣称支持。不要猜测未知字段。

## 本地解析

默认运行只输出消息数、跳过数和警告数，不输出正文：

```powershell
python scripts/import_weflow.py <导出文件>
```

只有用户明确要求生成本地证据文件时才提供 `--output`，并将输出放入 Git 仓库之外的私有数据目录。该文件只含文本证据消息、媒体数量和警告，不含系统消息或媒体占位正文：

```powershell
python scripts/import_weflow.py <导出文件> --output <私有目录>/derived/current/messages.json
```

脚本拒绝把 `--output` 写入检测到的 Git 工作区。解析不到任何可靠消息时以错误退出；无法归类的 TXT 行、非对象 JSON 项和缺失字段记录都会计入警告，不静默忽略。

## 提取顺序

1. 过滤系统与媒体占位消息。
2. 按发送者拆分用户消息和女朋友消息。
3. 从用户消息提取本人语气，从女朋友消息和互动序列提取人物证据。
4. 每个模式至少保留来源标识、情境和时间范围。
5. 一次性事件只能成为事实或待验证推测；多个独立情境重复后才升级为高置信度模式。

## 截图、照片和社交资料

- 截图：先转写可见原话并标明说话者；看不清时询问，不补写。
- 照片：只记录用户请求且有权使用的可见事实、日期或共同经历；不根据外貌判断性格。
- 社交资料：只处理公开发布或经授权提供的内容；公开人设与私下行为不自动视为一致。
