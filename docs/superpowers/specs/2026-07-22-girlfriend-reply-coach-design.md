# 哄女朋友 Skill 设计规格

## 目标

构建 `girlfriend-reply-coach`：针对一位具体的现任女朋友，根据经证据支持的人物档案和当前聊天情境，明确告诉用户怎么做，生成符合用户本人语气、可以直接发送的回复，并规划后续接话。

它不冒充女朋友，不使用通用“女生模板”，也不保证话术能控制对方反应。

## 架构

采用“通用 Skill 引擎 + Git 外私人人物档案”：

```text
Git 仓库
└─ girlfriend-reply-coach/
   ├─ SKILL.md
   ├─ agents/openai.yaml
   ├─ references/
   └─ scripts/

Git 外私有目录
└─ ~/.codex/private/girlfriend-reply-coach/
   ├─ self-voice.json
   ├─ people/<slug>/profile.json
   ├─ derived/<slug>/
   └─ raw/<slug>/             # 仅在用户明确要求保留时
```

Skill 更新不得覆盖私人档案。脚本拒绝把人物档案或聊天证据输出写入检测到的 Git 工作区。

## 输入与导入

支持当前聊天文字、截图、人工资料，以及 WeFlow 类 TXT、JSON、JSONL 文件。已验证 TXT 使用“时间 + 发送者、正文、空行”结构；JSON/JSONL 只接受文档列出的字段。ChatLab、Arkme JSON、HTML 等格式必须取得脱敏样例并验证字段后才宣称支持。

系统消息不进入证据；媒体占位只保留类型数量。无法识别可靠消息、发送者或字段时停止建档并输出警告。

## 人物与语气模型

人物结论分为：

- `已确认事实`：原话、实际行为或用户明确说明直接支持；
- `高置信度模式`：至少两个独立情境重复支持；
- `待验证推测`：证据少或有其他合理解释。

每条结论保存证据、适用情境、置信度、冲突关系和时间。新旧结论矛盾时双向保留冲突并降低置信度，不静默覆盖。

`self-voice.json` 独立记录用户称呼、句长、标点、emoji、安慰、道歉和撒娇表达，以及明确禁用表达。先决定关系策略，再用本人语气改写。

## 每次任务

1. 读取当前原话、背景和必要档案。
2. 区分已确认事实、高置信判断和待验证推测。
3. 判断她更可能需要倾听、理解、道歉、补救、陪伴、空间、边界或协商。
4. 明确给出用户现在要做的一至三步。
5. 除非用户只要分析，否则给一条可直接发送的主回复。
6. 说明为什么这样回、发送时机、等待窗口、雷区和不同回应下的下一句。
7. 收到新回复后延续上下文，不重新询问整套背景。

资料不足但能给低风险回复时先给回复；只有一个缺失信息会实质改变方向时，每轮只问一个关键问题。

## 隐私、更新和删除

- 导入前确认用户有权使用资料。
- 默认不保留原始文件，只保存必要派生信息。
- 私人资料不得进入 Git 或无关输出，也不得未经授权发送给第三方。
- 删除前预览绝对目标和确认令牌；确认后同时删除人物档案、派生摘要及该人物的保留原始副本。
- 删除后不得继续依赖已删除结论。

## 安全边界

拒绝 PUA、内疚操控、制造嫉妒、冷暴力、监控、秘密定位、翻手机、威胁、跟踪、冒充、欺骗、绕过拒绝和连续轰炸消息，并给诚实可执行的替代方案。

出现自伤、伤人、暴力、强迫或人身安全风险时，优先现实安全和求助渠道，不只生成安慰话术，也不用继续关系交换安全。

## 验收

- 15 个代表性行为场景有可复核输入、输出和判定工件。
- TXT、JSON、JSONL、畸形输入、证据过滤、Git 隔离、冲突档案和删除路径均有自动测试。
- 真实 WeFlow TXT 只输出统计即可成功解析，不保存正文。
- `quick_validate.py` 通过。
