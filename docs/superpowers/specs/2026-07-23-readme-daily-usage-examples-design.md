# README 日常调用与语气更新示例设计

## 目标

让刚生成对象专属 Skill 的用户可以直接照抄 README 完成两项操作：

1. 使用 `/hong-xiaoyu` 处理对方的一句话或多轮聊天；
2. 发现输出“不像我”时，重新学习本人语气并更新受影响的对象 Skill。

## 放置位置

两个示例紧接在“先建档，再生成每个人的专属 Skill”章节的生成流程之后、底层脚本说明之前。这样阅读顺序保持为“首次建档 → 日常调用 → 纠正更新 → 手动命令”。

## 日常调用示例

使用虚构对象代号 `xiaoyu`，同时展示：

- 单句输入：`/hong-xiaoyu 她说：“算了，你忙吧。”`
- 多轮聊天：在命令后原样粘贴双方消息；
- 输出约定：默认只返回一条完成策略分析和本人语气改写的可发送回复。

示例说明 `hong-xiaoyu` 必须已经生成；没有档案时不能把该命令当作临时对象切换。

## “不像我”更新示例

展示以下完整顺序：

1. 用户指出具体不符合本人习惯的表达；
2. 用户说“重新学习我的语气”；
3. 选择导入文件或粘贴经授权聊天；
4. 确认本人发送者映射并更新 `self-voice.json`；
5. 所有依赖旧本人语气的对象 Skill 被标记为待重新生成；
6. 重新预览摘要并明确确认；
7. 使用 `generate --force` 更新需要继续使用的对象 Skill。

更新示例不承诺静默自动覆盖，不把单次纠正直接提升为高置信度模式。

## 隐私边界

- 示例只使用虚构姓名和聊天。
- 导入新资料前仍须确认使用授权。
- README 不展示真实本地路径、真实发送者或档案内容。

## 验收标准

- README 包含 `/hong-xiaoyu` 的单句与多轮聊天示例。
- README 包含“不像我”和“重新学习我的语气”的完整更新流程。
- README 明确说明更新后需要重新预览、确认并生成。
- README 契约测试锁定上述关键内容。
