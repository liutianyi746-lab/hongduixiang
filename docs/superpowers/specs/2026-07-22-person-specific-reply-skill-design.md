# 针对个人的专属回复 Skill 设计

## 目标

将 `girlfriend-reply-coach` 从单一回复 Skill 扩展为“专属回复 Skill 生成器”：用户先提供与特定对象的经授权聊天记录，系统分别学习对象的互动模式和用户本人的表达习惯，再生成一个只针对该对象、本地可调用的子 Skill。

首个实例为呆佳，生成的本地 Skill 名称为 `hong`，固定绑定私有档案 `daijia`。

## 两层结构

### 通用生成器

公开仓库中的 `girlfriend-reply-coach` 负责：

1. 导入 TXT、JSON 或 JSONL 聊天记录；
2. 要求用户确认资料使用授权；
3. 确认“我”和“对象”的发送者映射；
4. 分别提取对象档案和本人语气档案；
5. 生成本地专属子 Skill；
6. 支持后续纠正、更新、删除和重新绑定。

通用生成器不得包含任何真实人物资料。

### 本地专属子 Skill

生成的 `hong` 负责日常回复，固定读取：

- `~/.codex/private/girlfriend-reply-coach/people/daijia/profile.json`；
- `~/.codex/private/girlfriend-reply-coach/self-voice.json`；
- 当前聊天或事件背景；
- 当前问题必要的派生摘要。

它不复制私人画像到 Skill 目录，只保存绑定标识和读取规则。

## 建档流程

### 1. 授权与本地解析

- 导入前确认用户有权使用资料。
- 默认在本地解析；不向第三方发送资料。
- 默认不复制原始聊天文件。
- 只保存必要的统计、证据索引、短摘要和结论。

### 2. 双方映射

解析完成后必须展示发送者及消息数，让用户确认：

- 哪个发送者是用户本人；
- 哪个发送者是目标对象；
- 目标对象的代号和本地 slug。

未确认映射前不得建立人物或本人语气结论。

### 3. 对象档案

从对象消息和双方互动序列提取：

- 称呼、句长、标点和亲密表达；
- 可观察的情绪表达与冲突行为；
- 偏好、雷区、重要日期和共同经历；
- 倾听、陪伴、空间、解释和补救等互动需要；
- 曾经有效或无效的回复方式。

只记录有证据的可观察模式。星座、MBTI、单句话、回复速度或词频不能单独证明人格、动机、依恋类型、忠诚度或爱意。

### 4. 本人语气档案

只从用户本人消息和用户明确纠正中提取：

- 常用称呼；
- 单条长度、短句连发和段落习惯；
- 语气词、标点、表情和中英文混用；
- 道歉、安慰、解释、撒娇和表达爱意的方式；
- 用户明确不会说的措辞；
- 不同情境下的表达差异。

不得把对象的说话方式混入本人语气档案。

## 回复生成

运行时严格分成两步：

1. 根据对象档案、关系证据和当前情境决定“此刻应该表达什么”；
2. 根据本人语气档案决定“用户真实会怎么表达”。

默认交付：

- 一条可直接发送的主回复；
- 一至三项当前动作；
- 发送时机与等待窗口；
- 对方积极、含糊、拒绝或不回复时的下一步。

不得冒充对象、替对象发言、虚构承诺或保证回复一定有效。

## 简短调用

自然语言触发：

```text
哄：算了你忙吧
```

Codex 强制调用：

```text
$hong 算了你忙吧
```

Claude Code 强制调用：

```text
/hong 算了你忙吧
```

`hong` 默认绑定呆佳。只有用户明确要求换对象、创建新对象或当前绑定不存在时才询问人物。

## 持续纠正

用户说“不像我”时，区分并更新本人语气档案；用户说“她不会这样”时，区分并更新对象事实或互动判断。

新证据与旧结论冲突时：

1. 保留双方证据；
2. 建立冲突链接；
3. 降低相关置信度；
4. 标记待验证；
5. 从下一条回复开始采用已确认纠正。

## 文件边界

公开仓库：

```text
girlfriend-reply-coach/
├── SKILL.md
├── scripts/
├── references/
└── templates/hong/SKILL.md
```

本地私有目录：

```text
~/.codex/private/girlfriend-reply-coach/
├── manifest.json
├── self-voice.json
├── people/<slug>/profile.json
├── derived/<slug>/
└── generated-skills/<skill-name>/binding.json
```

本地安装目录：

```text
~/.codex/skills/hong/
└── SKILL.md
```

Claude Code 使用相同模板安装到 `~/.claude/skills/hong/`。绑定文件只存代号和私有路径，不保存聊天正文或画像副本。

## 错误与降级

- 没有聊天记录：只能提供明确标记的通用临时回复，不宣称个性化。
- 映射未确认：停止建档，只展示发送者统计。
- 私有档案不存在：提示重新绑定或恢复档案，不猜测人物。
- 当前背景不足：先给低风险临时回复，每轮最多追问一个会改变方向的问题。
- 危险、强迫或操控情境：优先安全和边界，不生成操控话术。

## 验收标准

- 通用 Skill 明确要求先建双人档案，再宣称个性化。
- 提供可复用的专属子 Skill 模板和本地生成命令。
- `hong` 能固定绑定 `daijia`，并支持 `哄：`、`$hong` 和 `/hong` 三种入口。
- 私有档案不复制到 Git 或公开 Skill 目录。
- 缺失绑定、映射错误和未建档状态均有明确错误或降级行为。
- 现有导入、画像、隐私、安全与 15 类行为场景测试保持通过。
- 新增双人映射、专属 Skill 生成、绑定读取与短触发测试。
