# README 个性化首屏文案设计

## 目标

让访问者在 README 开头立即理解：本项目不是固定话术库，而是根据经授权的真实聊天记录，对具体对象和具体情境进行个性化、差异化回复生成。

## 文案结构

标题后依次放置：

1. 保留一句产品定位：面向 Codex 和 Claude Code CLI 的个性化「哄对象」回复 Skill。
2. 使用选定的价值主张：不再套用千篇一律的话术；从聊天记录提取对象偏好、情绪模式和沟通雷区，同时学习用户本人的称呼、句长、语气词与表达习惯，针对具体对象和当前情境差异化生成可直接发送的回复。
3. 用现有安全提示强调真实沟通，不操控、监视或冒充。

## 边界

- 使用“经授权的聊天记录”，不暗示可以秘密获取他人资料。
- 不声称完全复刻真人、保证回复有效或自动改变关系。
- 不把本地处理写成绝对保证；具体隐私机制仍由 README 后文说明。
- 只修改 README 首屏文案，不调整 Skill 行为、导入器或档案模型。

## 验收标准

- README 前 8 行包含“聊天记录”“具体对象”“当前情境”“差异化生成”和“符合你本人语气”。
- 首屏明确不是千篇一律的哄人话术。
- 保留不操控、不监视、不冒充和不替代真实交流的提示。
- Markdown 检查和全部自动化测试通过。
