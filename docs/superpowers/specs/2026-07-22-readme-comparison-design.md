# README 组合优势章节设计

## 目标

在根目录 `README.md` 的“工作方式”之后增加一节，解释项目为何结合 `goutoujunshi` 的关系判断能力与 `create-ex` 的人物蒸馏方法，以及这种组合相对常见替代方案的具体优势。

## 核心表述

- `goutoujunshi` 贡献“说什么”：情绪承接、事实与推测分离、关系利益判断、边界、行动和后续接话。
- `create-ex` 贡献“如何从材料认识一个人”：从聊天和用户纠正中提取关系记忆、表达习惯与互动模式，并持续更新。
- 本项目不模拟或冒充现任对象，而是将人物蒸馏方法改造为对象档案和用户本人语气档案。
- 最终流程是先判断正确的沟通目标，再用用户有证据支持的真实口吻生成回复。

## 对比方式

采用一张紧凑表格，对比以下五类方案：

1. 通用聊天机器人；
2. 固定哄人话术库；
3. 纯语气模仿；
4. 单一恋爱军师；
5. 本项目。

对比维度包含：当前情境判断、对象个性化、用户本人语气、证据约束、持续纠正、行动与后续、隐私设计。

## 表达边界

- 不声称某条回复一定能改变对方、消除冲突或挽回关系。
- 不贬低参考 Skill；明确本项目是复用其方法论，而非直接把前任模拟功能用于现任。
- 不把 MBTI、星座、性别或单条消息作为人格结论。
- 不泄露真实人物、聊天记录或用户私有统计。
- 优势以可验证的设计能力描述，不使用“最强”“百分百有效”等营销措辞。

## 验收标准

- 新章节位于“工作方式”和“安装”之间。
- 同时出现 `goutoujunshi`、`create-ex` 及其实际分工。
- 表格覆盖五类方案与七个比较维度。
- 清楚解释“先决定说什么，再决定怎么说”。
- 包含效果边界与不冒充真人的说明。
- README Markdown 结构检查及现有 18 项测试全部通过。
